# DuckDuckWash v1.0.4 — Aunt Feedback Round 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship five aunt-requested features — editable order date, half-step SOCKS quantity, per-section custom clothing input, iron default price of 40, and a per-order urgent surcharge percentage.

**Architecture:** SQLite-backed Electron app. Pure business logic in `src/main/logic/*` (unit-tested with vitest), schema + migrations in `src/main/db.ts`, IPC wiring in `src/main/ipc.ts`, and a single detail screen `src/renderer/src/screens/OrderDetails.tsx` for the UI. Surcharge math lives in `pricing.ts`; everything else flows through the existing `saveDetails` IPC.

**Tech Stack:** Electron, React 19, TypeScript, better-sqlite3 (WAL), Tailwind + daisyUI, vitest.

**Spec:** `docs/superpowers/specs/2026-06-16-v1.0.4-aunt-feedback-design.md`

```ts
export function computeOrderTotal(items: PricedItem[], surchargePct = 0): number {
  if (items.length === 0) throw new Error('order must have at least one item')
  for (const it of items) {
    if (it.quantity <= 0) throw new Error('quantity must be positive')
    if (it.unit_price <= 0) throw new Error('unit price must be positive')
  }
  if (surchargePct < 0) throw new Error('surcharge must not be negative')
  const subtotal = items.reduce((acc, it) => acc + it.quantity * it.unit_price, 0)
  return subtotal * (1 + surchargePct / 100)
}
```

## Task 5: saveDetails IPC — surcharge, date edit, half-step guard

**Files:**
- Modify: `src/main/ipc.ts`

- [ ] **Step 1: Replace the saveDetails handler**

Replace the entire `ipcMain.handle('orders:saveDetails', …)` block (currently lines 55-83) with:

```ts
  ipcMain.handle('orders:saveDetails', (_e, input: OrderDetailsInput) => {
    const order = db.prepare('SELECT * FROM orders WHERE id=?').get(input.order_id) as { is_delivery: number; status: OrderStatus } | undefined
    if (!order) throw new Error('order not found')
    if (order.status === 'closed') throw new Error('order is closed')
    if (input.garments.length === 0) throw new Error('add at least one garment')
    for (const g of input.garments) if (g.quantity <= 0) throw new Error('garment quantity must be positive')
    const surchargePct = input.surcharge_pct ?? 0
    if (surchargePct < 0) throw new Error('surcharge must not be negative')
    const total = computeOrderTotal(
      input.items.map((i) => ({ quantity: i.quantity, unit_price: i.unit_price })),
      surchargePct
    )
    // optional date edit; noon localtime matches the backdate convention
    const createdAt = input.created_at && /^\d{4}-\d{2}-\d{2}$/.test(input.created_at)
      ? `${input.created_at} 12:00:00`
      : null
    const tx = db.transaction(() => {
      const upd = db.prepare('UPDATE order_items SET quantity=?, unit_price=?, total=? WHERE id=? AND order_id=?')
      for (const i of input.items) upd.run(i.quantity, i.unit_price, i.quantity * i.unit_price, i.item_id, input.order_id)

      db.prepare('DELETE FROM order_garments WHERE order_id=?').run(input.order_id)
      const insG = db.prepare(
        'INSERT INTO order_garments (order_id, garment, quantity, special_care, wearer) VALUES (?,?,?,?,?)'
      )
      const wearers = ['male', 'female', 'child']
      for (const g of input.garments)
        insG.run(input.order_id, g.garment, g.quantity, g.special_care ? 1 : 0,
          wearers.includes(g.wearer) ? g.wearer : 'female')

      const newStatus = order.status === 'waiting_input' ? 'in_progress' : order.status
      db.prepare(
        `UPDATE orders SET total=?, status=?, is_delivery=?, surcharge_pct=?,
         created_at=COALESCE(?, created_at) WHERE id=?`
      ).run(total, newStatus, input.is_delivery ? 1 : 0, surchargePct, createdAt, input.order_id)
    })
    tx()
    return total
  })
```

Changes from the original: garment guard `< 1` → `<= 0` (allows SOCKS 0.5); compute total with surcharge; persist `surcharge_pct`; `created_at=COALESCE(?, created_at)` so a null leaves the date untouched.
